# dot 文档

## 当前设计

[`design-baseline.md`](design-baseline.md) 是 `dot` 重设计后唯一具有规范性的产品与行为基线。
它规定 MVP 的用户配置、公开命令、所有权、清理边界和明确接受的风险，但不证明当前代码已经
实现这些能力。

仓库根 [`README.md`](../README.md)、代码和测试说明当前实现状态。重写完成前，当前实现与新
基线存在差异是已知过渡状态，不得把目标设计描述为已经可用。

## 历史存档

[`archive/`](archive/) 保存重设计前的规范、实现说明和工程计划，只用于追溯旧实现。历史内容
不再参与规范裁决或当前工作流程，也不得作为新增代码或兼容行为的依据。

## 文档原则

- 当前产品契约保持单一真相源，不再拆成多篇互相引用的安全规范。
- 内部 package、类型、算法、系统调用和测试组织由实现与测试决定。
- 新能力或防御只有在真实用户故事、已发生故障或不可接受的数据损失路径能够证明收益时才
  加入；“理论上更完整或更安全”不构成立项理由。
